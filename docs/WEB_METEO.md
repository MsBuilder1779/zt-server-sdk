# Web Meteo в ZT Server SDK

Этот документ описывает использование Web Meteo через готовый Android SDK.

Файл SDK:

```text
libs/zt-server-sdk-0.2.0.aar
```

Исходный код SDK в репозитории не публикуется. Документ содержит только информацию, которая нужна для подключения и использования SDK в Android-приложении.

## Назначение

Web Meteo через ZT Server SDK позволяет Android-приложению получать метеоданные через серверную экосистему ZEvS.

SDK берёт на себя сетевой слой, обработку результата, обработку ошибок и работу со служебными данными. Приложению не нужно вручную собирать служебные запросы или передавать внутренние параметры SDK.

## Возможности

Web Meteo SDK предоставляет:

- получение метеоданных по координатам;
- передачу высоты точки;
- выбор типа результата;
- поддержку запроса на конкретное время, если это предусмотрено текущей версией SDK;
- обработку успешного ответа;
- обработку ошибок сети;
- обработку ошибок доступа;
- обработку ошибок конфигурации;
- отделение Web Meteo доступа от обычной авторизации приложения.

## Подключение SDK

Скопируйте AAR-файл в Android-приложение:

```text
app/libs/zt-server-sdk-0.2.0.aar
```

Добавьте зависимость в `app/build.gradle`:

```gradle
dependencies {
    implementation files("libs/zt-server-sdk-0.2.0.aar")

    implementation "org.jetbrains.kotlinx:kotlinx-coroutines-android:1.9.0"
    implementation "org.jetbrains.kotlinx:kotlinx-serialization-json:1.7.3"
    implementation "com.squareup.okhttp3:okhttp:4.12.0"
}
```

Если эти зависимости уже есть в проекте, повторно добавлять их не нужно.

## Разрешение на интернет

Добавьте разрешение в `AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## Инициализация SDK

Инициализируйте SDK один раз при запуске приложения, например в `Application` или в основной точке старта приложения:

```kotlin
import com.zevsecosystem.serversdk.ZevsSdk
import com.zevsecosystem.serversdk.core.ZevsSdkConfig

ZevsSdk.init(
    context = applicationContext,
    config = ZevsSdkConfig(
        baseUrl = "https://zevs-team.ru",
        appVersionCode = BuildConfig.VERSION_CODE,
        appVersionName = BuildConfig.VERSION_NAME
    )
)
```

## Использование Web Meteo

Типовой сценарий:

1. Приложение получает координаты точки.
2. Приложение получает или задаёт высоту точки.
3. Приложение выбирает нужный тип результата.
4. Приложение передаёт Web Meteo token через публичный механизм SDK, предназначенный для Web Meteo.
5. Приложение вызывает публичный метод SDK для получения метеоданных.
6. Приложение показывает результат пользователю или обрабатывает ошибку.

Пример входных данных:

```text
latitude: 55.75
longitude: 37.61
height: 180.0
```

Если приложение поддерживает запрос прогноза на конкретное время, используйте публичный параметр SDK для времени. Не формируйте серверный запрос вручную.

## Web Meteo token

Web Meteo token используется только для Web Meteo.

Не используйте его как обычный токен авторизации приложения.

Не смешивайте Web Meteo token с обычной регистрацией, авторизацией и защищёнными запросами приложения.

## Правила использования

Приложение не должно вручную передавать:

- идентификатор устройства;
- ключ приложения;
- access token;
- refresh token;
- служебные заголовки авторизации;
- внутренние параметры SDK.

SDK управляет этими данными самостоятельно.

## Обработка ошибок

Приложение должно быть готово к таким ситуациям:

- Web Meteo token не задан;
- Web Meteo token неверный;
- нет подключения к интернету;
- сервер временно недоступен;
- координаты некорректны;
- высота не задана или задана неверно;
- ответ сервера не может быть обработан;
- доступ временно ограничен.

Рекомендуется показывать пользователю понятные сообщения, а не технический текст ошибки.

Примеры пользовательских сообщений:

```text
Нет подключения к интернету.
Не удалось получить метеоданные.
Проверьте координаты точки.
Сервис временно недоступен.
Требуется обновить доступ к Web Meteo.
```

## Что не нужно делать

Не нужно:

- вручную собирать серверный URL;
- вручную добавлять служебные заголовки;
- вручную подставлять внутренние идентификаторы;
- хранить служебные токены вне SDK;
- смешивать Web Meteo доступ с обычной авторизацией приложения;
- раскрывать пользователю внутренние детали запроса.

## Проверка подключения

После подключения SDK выполните сборку приложения:

```powershell
./gradlew assembleDebug
```

На Windows CMD:

```bat
gradlew.bat assembleDebug
```

Если приложение собирается, SDK подключён корректно.

## Минимальный чек-лист

Перед использованием Web Meteo проверьте:

- SDK AAR лежит в `app/libs`;
- зависимость `implementation files(...)` добавлена;
- зависимости Kotlin Coroutines, Serialization и OkHttp доступны;
- разрешение `INTERNET` добавлено;
- SDK инициализирован при старте приложения;
- Web Meteo token передаётся через публичный механизм SDK;
- координаты и высота точки корректны;
- приложение обрабатывает ошибки.

## Безопасность

Репозиторий содержит только готовый AAR-файл SDK.

Основная проверка доступа, права пользователя, привязка устройства, проверка токенов и ограничения выполняются на стороне сервера.

SDK не требует раскрытия внутренних технических деталей для подключения и использования Web Meteo.
